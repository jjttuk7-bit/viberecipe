## 잔존 위험 / 가드

- **R-BR-1**: 대화 스크롤 영역에서 *과거 산출물 카드*가 무한 누적되는 위험 — 본 사이클은 최신 응답의 임베드만 표시(BuildMode가 lastResponse 1개 보존하던 패턴 유지). 과거 메시지의 카드는 *접힘* 또는 *제거*.
- **R-BR-2**: 옵션 칩 1~3개는 LLM 응답에 의존 — 0개 케이스에 대비해 입력 행만으로도 진행 가능해야 함.
- **R-BR-3**: ENGINE.md `options` 최대 15자 제약 — 칩 UI가 텍스트 오버플로 없이 표시 가능해야 함.
- **R-BR-4**: 게이지 시각화 — TASTE §1 6축 + 식감 5축 = 11축. 한 화면에 모두 표시 가능해야 함 (gauges 그리드 2~3 col).
- **R-BR-5**: 사용자가 보낸 user 메시지가 *왜 보내졌는지* 명확해야 함 — 칩 탭 ↔ 자유 텍스트 입력 둘 다 user 메시지 본문이 됨. 둘 시각 차이 없어도 무방(보낸 결과는 동일).
