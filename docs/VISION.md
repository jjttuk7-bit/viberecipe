## 0. 한 줄

자연어로 페어 셰프와 대화하며 **나만의 레시피를 같이 빌드하고(과정) · 실제로 요리해 결과를 되먹이는(결과)** 요리 운영체제. "vibe-X" 패밀리의 음식판.

---

## 1. vibe-X 논제

바이브 코딩/뮤직의 공식 = **자연어 대화로 같이 만들고(과정 만족) → 실제로 작동하는 결과물(결과 만족)**. 두 만족을 동시에 준다.

음식이 vibe-X 중 가장 강한 이유:
- 매일·하루 3번·전 인류 (TAM 비교 불가)
- 결과 검증이 즉각·감각적 ("맛있나")
- 컴파일 타깃이 *너의 입맛·부엌·몸* → 개인화가 본질, 학습 해자가 깊어짐
- 멀티모달·습관·소셜 천성

---

## 2. 해자 (복제 불가능성)

ChatGPT도 레시피는 준다. 우리의 *유일한* 차별점은 헌법 §1·§4의 그것:

> **빌드 → 실제로 요리(런타임) → 결과 되먹임 → 부엌 지문 학습** 의 끊기지 않는 루프.

이게 없으면 예쁜 레시피 봇, 있으면 *쓸수록 나만 알아주는 요리 OS*. **모든 결정은 "이 루프를 더 끈끈하게 하는가"로 판단한다.**

추가 해자: 유케이의 도메인 권위(로스팅/관능평가 + K-Food Formula 시스템) = 카피 불가한 취향 해자(D-009).

## 3. 층위 아키텍처 (★ 핵심 — 절대 무너뜨리지 말 것)

| 층 | 무엇 | 성격 |
|---|---|---|
| **L1 — 코어 (보편)** | 에이전트 OS: 대화 빌드 → 요리(런타임) → 패치 → 학습. 과정+결과 만족. | **요리권 무관(cuisine-agnostic)**. 특정 음식문화에 종속 금지. |
| **L2 — 지식 팩 (플러그인)** | 교체 가능한 콘텐츠 모듈. **K-Food Formula = 첫 번째 팩**. 이후 이탈리안·일식·베이킹·"당신만의 공식" 등 추가. | `lib/formulas/` 데이터로 끼움. 코어는 그대로. |
| **L3 — 경험/도달 (확장)** | vibe 정체성, 모바일·핸즈프리, 멀티모달, 글로벌/i18n, 재미·소셜, vibe-X 패밀리. | 보편적 야망. |

### 가드레일 — "책은 팩이지 플랫폼이 아니다"
- **스키마/엔진(L1)에 K-food(또는 어떤 단일 요리권) 가정을 박지 않는다.**
- K-Food Formula는 셰프를 똑똑하게 만드는 *첫 지식 주입*이자 시장 진입 wedge일 뿐, 앱이 그것만 하는 게 아니다.
- **덮이면 안 되는 것들**(항상 1순위): 보편적 vibe 경험 / 멀티 요리권 / 글로벌 / OS 루프 / 개인화 / 재미·소셜.

## 4. K-Food Formula Kitchen = 첫 L2 팩

유케이의 책 『K-Food Formula Kitchen』(12 공식, KR/EN 이중언어, Etsy 판매)은 한식 도메인을 시스템화한 자산. **정적(책) ↔ 라이브(앱)** 같은 비전의 두 형태.

| 책 | 앱(Vibe Recipe) |
|---|---|
| 공식 고르기 (Base→Balance→Build→Boost) | concept 단계 |
| 실전 비율 (예: 김치찌개 김치1:국물½:마늘1t:설탕1t:물2컵) | base 기본 구성(기본-먼저) |
| 재료 → 맛 역할 (간장=짠/감칠, 고추장=맵/단…) | taste 게이지 엔진(근거 있는 조절) |
| 예시 레시피 | steps |
| 입맛 조절 + Beginner Tip ("시면 설탕+졸이기") | 런타임 패치/디버깅 |

"공식이 기준을 만들고, 당신의 입맛으로 완성" = 우리 논제 그 자체.

## 5. 시장 / wedge

- **비치헤드**: "요리 모르는데 한식 만들고 싶은 초보" — 국내 자취·1인가구 + **해외 K-food 초보**(한식 글로벌 상승세 + 책 이중언어). 포뮬러+페어셰프가 정확히 이들을 위한 것.
- 확장: 식단관리(알러지·비건·당뇨)·홈파티·멀티 요리권.
- 퍼널 후보: 책(Etsy) 리드마그넷 → 앱 라이브/구독. 또는 앱이 "당신만의 포뮬러" 생성으로 책 확장.

## 6. 페이즈 (잠정)

- **P-A 단일 플레이어 풀루프**: BUILD→COOK→회고→학습을 *모바일*에서 완성 + 계정/영속. (지금 COOK 없음 = 최우선 갭)
- **P-B 멀티모달·안전·i18n**: 냉장고 사진, 식이/알러지 안전, 현지화(재료·단위·요리권).
- **P-C 소셜·집단지성**: 공유·포크, n명 실패 → 레시피 자가진화(네트워크 효과, 헌법 P3).
